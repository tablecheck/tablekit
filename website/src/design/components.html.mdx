# Design Foundations

[Alerts](/design/components/alerts)

Alerts to notify the user of events, communication actions that are required or that have taken place.

[Badges](/design/components/badges)

A badge, sometimes called a tag or label, is a UI atom which indicates a status, notification, or event that appears in relativity with the underlying object.

[Buttons](/design/components/buttons)

Buttons are clickable elements that are used to trigger actions. They communicate calls to action to the user and allow users to interact with pages in a variety of ways. Button labels express what action will occur when the user interacts with it.

[Button group WIP](/design/components/button-group)

Button groups are used to select or toggle between different grouped options where there are a fixed number. They should not be used when the number of options are variable or need to scale indefinitely.

[Calendar & Date Picker WIP](/design/components/calendar-date-picker)

Date pickers and Time pickers allow selecting a single value from a pre-determined set.

[Checkbox WIP](/design/components/checkbox)

Checkboxes are used when there are multiple items to select in a list. Users can select zero, one, or any number of items.

[Chips WIP](/design/components/chips)

Chips allow users to enter information, make selections, filter content, or trigger actions. While buttons are expected to appear consistently and with familiar calls to action, chips should appear dynamically as a group of multiple interactive elements.

[Dialog WIP](/design/components/dialog)

Dialogs inform users about a task and can contain critical information, require decisions, or involve multiple tasks.

[Icon button WIP](/design/components/icon-button)

A icon button is a icon which triggers an action on click, they communicate calls to action to the user and allow users to interact with pages in a variety of ways.

[Input WIP](/design/components/input)

Allows a user to enter data, edit data and search.

[Phone input WIP](/design/components/phone-input)

Phone input allows users to easily input and validate a phone number.

[Radio button WIP](/design/components/radio-button)

Use radio buttons when you have a group of mutually exclusive choices and only one selection from the group is allowed.

[Select WIP](/design/components/radio)

Allows user to select, search, and interact with a list of items.

[Select list WIP](/design/components/select-list)

Select lists are used along with buttons and select inputs to choose one or multiple options.

[Tabs WIP](/design/components/tabs)

Tabs are used to organize related content. They allow the user to navigate between groups of information that appear within the same context.

[Toggle WIP](/design/components/toggle)

Toggle is a control that is used to quickly switch between two possible states. Toggles are only used for these binary actions that occur immediately after the user “flips the switch”. They are commonly used for “on/off” switches.
