# Chips WIP

<IntroText>
  Chips allow users to enter information, make selections, filter content, or
  trigger actions. While buttons are expected to appear consistently and with
  familiar calls to action, chips should appear dynamically as a group of
  multiple interactive elements.
</IntroText>

#### Figma: [Chips](<https://www.figma.com/file/mTd42WpVg1Ehjl3GImVKNS/%5BPublic%5D-TableKit-Components-(3.0)?node-id=4006%3A51372>)

## [Rules](website/src/design/components/chips.html#rules)

<TwoCol>

<div>
  <ImageWrap>

![Rules](/static/img/components/chip-rules-5.svg 'Button rules #5')

  </ImageWrap>

  <Rule>
    Use chips when there are at least 2 options.
  </Rule>
</div>

<div>
  <ImageWrap>

![Rules](/static/img/components/chip-rules-6.svg 'Button rules #6')

  </ImageWrap>

  <Rule data-variant='error'>
   Don't use when there is a single option present.
  </Rule>
</div>

<div>
  <ImageWrap>

![Rules](/static/img/components/chip-rules-1.svg 'Button rules #1')

  </ImageWrap>

  <Rule data-variant='error'>
    Don't stack chips vertically
  </Rule>
</div>
<div>
  <ImageWrap>

![Rules](/static/img/components/chip-rules-2.svg 'Button rules #2')

  </ImageWrap>

  <Rule data-variant='warning'>
    When space is limited, horizontal scrolling is OK on mobile/touch devices, stacking rows is OK on desktop.
  </Rule>
</div>

<div>
  <ImageWrap>

![Rules](/static/img/components/chip-rules-7.svg 'Button rules #3')

  </ImageWrap>

  <Rule>
    Use them as toggles, where only one option can be selected.
  </Rule>
</div>

<div>
  <ImageWrap>

![Rules](/static/img/components/chip-rules-8.svg 'Button rules #4')

  </ImageWrap>

  <Rule>
   Use them as multi-selects, where multiple or all options can be selected.
  </Rule>
</div>
<div>
  <ImageWrap>

![Rules](/static/img/components/chip-rules-3.svg 'Button rules #3')

  </ImageWrap>

  <Rule data-variant='warning'>
    Try to use at least 3 characters to avoid losing the structure of the chips.
  </Rule>
</div>

<div>
  <ImageWrap>

![Rules](/static/img/components/chip-rules-4.svg 'Button rules #4')

  </ImageWrap>

  <Rule data-variant='error'>
   Use overly long labels, particularly repeated prefix or suffix text.
  </Rule>
</div>
