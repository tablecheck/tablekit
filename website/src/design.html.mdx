# Design Guidelines

TableKit's interface guidelines are a collection of principles, standards, and usage guidelines for designing TableCheck interfaces.

[Foundations](/design/foundations)

The fundamental parts of the design system that underpin all TableCheck interfaces, such as color and typography.

[Accessibility](/design/accessibility)

Standards, guidelines, and tools to design accessible TableCheck interfaces.

[UI patterns](/design/ui-patterns)

Design guidelines covering common user workflows.

[Components](/design/components)

Design and usage guidelines for individual TableKit components.

[Using Figma](/design/using-figma)

A guide on how to get started using Figma to design interfaces at TableCheck.

Need help?
If you found a bug on this website, please [open a new issue](https://github.com/tablecheck/tablekit/issues/new) with as much detail as possible, including steps to replicate the bug, screenshots, links, and expected results.

If you need help using and designing with TableKit, and you are TableCheck staff, please visit the #tablekit Slack channel.
